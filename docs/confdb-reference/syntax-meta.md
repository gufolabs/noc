# ConfDB Syntax: meta { #meta }

The `meta` section exposes various Managed Object's database attributes to configuration tree.

## meta id { #meta-id-id }

Managed object's id.

```
meta id 777
```

## meta name { #meta-name-name }

Managed object's name.

```
meta name sw1
```

## meta description { #meta-description-description }

Managed object's description.

```
meta description "Don't reboot this router"
```

## meta profile { #meta-profile-profile }

NOC SA profile.

```
meta profile Juniper.JUNOS
```

## meta vendor { #meta-vendor-vendor }

Equipment vendor.

```
meta vendor Juniper
```

## meta platform { #meta-platform-platform }

Equipment platform.

```
meta platform mx480
```

## meta version { #meta-version-version }

Software version

```
meta version 9.6R3
```
